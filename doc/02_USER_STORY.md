# ユーザーストーリー - VanishToDo

## 1. ユーザーペルソナ

### プライマリーペルソナ：大河るり
- **役割**: 開発者兼唯一のユーザー
- **背景**: 
  - ~~本業を持ちながら~~個人開発を行っている
  - 複数のタスクを同時並行で管理する必要がある
  - 従来のToDoアプリを使用したが継続できなかった経験がある
- **ニーズ**:
  - タスクを頭から出して整理したい
  - 簡単に素早くタスクを記録したい
  - タスクの山に圧倒されたくない
  - 日々の達成感を感じたい
- **ペインポイント**:
  - タスク入力が面倒
  - 未完了タスクの蓄積によるストレス
  - ToDoアプリを開くのが億劫になる

## 2. エピック（大項目）

### エピック1: タスク入力の簡素化
最小限の入力で充実したタスク情報を作成できる

### エピック2: タスク表示の最適化
心理的負担を軽減し、達成感を得やすい表示システム

### エピック3: タスク管理の柔軟性
状況に応じて適切なタスク管理ができる

### エピック4: システムの持続可能性
長期的に使い続けられるシステム

## 3. ユーザーストーリー詳細

### エピック1: タスク入力の簡素化

#### US-1.1: クイックタスク入力
**As a** ユーザー  
**I want to** 思いついたタスクを即座に入力できる  
**So that** タスクを忘れる前に記録できる

**受け入れ条件:**
- [ ] アプリを開いてすぐにタスク入力欄にフォーカスが当たる
- [ ] Enterキーもしくは登録ボタンでタスクを追加できる
- [ ] 入力後、自動的に入力欄がクリアされ次の入力ができる
- ~~[ ] 5秒以内にタスクの入力から保存まで完了できる~~

#### US-1.2: 自然言語でのタスク入力
**As a** ユーザー  
**I want to** 曖昧な表現や省略した文でタスクを入力できる  
**So that** 詳細を考えずに素早く入力できる

**受け入れ条件:**
- [ ] 「メール」と入力し解析ボタンを押したら「メールを確認する」のように補完される
- [ ] 「明日締切のレポート」と入力し解析ボタンを押したら締切日が自動設定される
- [ ] 「買い物 牛乳 パン」のような箇条書きも~~理解される~~文に変換される
- [ ] LLMによる補完結果をプレビューで確認できる

#### US-1.3: メモからタスク分割
**As a** ユーザー  
**I want to** 長文のメモを複数のタスクに自動分割できる  
**So that** 思考を整理しながらアクショナブルなタスクを作成できる

**受け入れ条件:**
- [ ] ~~複数行のテキストを入力できる~~長文を入力できる
- [ ] 「~~分割~~解析」ボタンでLLMがタスクを分析・分割する
- [ ] 分割されたタスクをプレビューで確認できる
- [ ] 不要なタスクを削除してから一括追加できる
- [ ] 各タスクの重さ（重・中・軽）が自動判定される

#### US-1.4: タスクの重さ自動判定
**As a** ユーザー  
**I want to** タスクの重さを自動的に判定してもらえる  
**So that** 重さの設定に悩まずに済む

**受け入れ条件:**
- [ ] 解析ボタンを押すとタスク内容から推定所要時間を判定
- [ ] 重（1時間以上）、中（15分〜1時間）、軽（15分以内）に分類
- [ ] 判定結果を手動で変更可能
- ~~[ ] よく使うタスクパターンを学習して精度が向上する~~

### エピック2: タスク表示の最適化

#### US-2.1: 制限付きタスク表示
**As a** ユーザー  
**I want to** 今やるべきタスクだけを見る  
**So that** タスクの山に圧倒されずに済む

**受け入れ条件:**
- [ ] デフォルトでは各重さカテゴリで3-5個程度のタスクのみ表示
- [ ] 古いタスクは自動的に非表示になる
- [ ] 締切ありタスクは全て表示される
- [ ] 「もっと見る」オプションで追加表示可能

#### US-2.2: 重さ別フィルタリング
**As a** ユーザー  
**I want to** タスクの重さでフィルタリングできる  
**So that** 今の気分や時間に合わせてタスクを選べる

**受け入れ条件:**
- [ ] 画面下部に「すべて」「重」「中」「軽」「締切」のフィルタボタン
- [ ] ワンタップでフィルタ切り替え可能
- [ ] 現在の選択状態が視覚的に分かる
- [ ] フィルタ状態はセッション間で保持される

#### US-2.3: 日次達成モード
**As a** ユーザー  
**I want to** その日のノルマを達成したら追加タスクを表示しない  
**So that** 達成感を味わい、休息できる

**受け入れ条件:**
- [ ] 日次目標タスク数を設定できる（例：軽3個、中2個、重1個）
- [ ] 目標達成したら「今日はお疲れ様」メッセージ表示
- [ ] 「もっとやる」ボタンで追加タスクを表示可能
- [ ] 翌日になると自動的にリセットされる

#### US-2.4: タスク完了フィードバック
**As a** ユーザー  
**I want to** タスクを完了した時に気持ちの良いフィードバックを得る  
**So that** 達成感を感じられる

**受け入れ条件:**
- [ ] ボタンをタップorクリックで完了マーク
- ~~[ ] 完了時に軽快なアニメーション~~
- [ ] 完了タスクは~~一時的に表示後、~~フェードアウト
- [ ] 本日の完了タスク数がどこかに表示される(n/m)

### エピック3: タスク管理の柔軟性

#### US-3.1: タスク編集
**As a** ユーザー  
**I want to** 既存のタスクを編集できる  
**So that** 状況の変化に対応できる

**受け入れ条件:**
- [ ] タスクが表示されているときではいつでもタスクをタップして編集モードに入れる
- [ ] タスク名、重さ、締切を変更可能
- [ ] 編集中も他のタスクは操作可能（モーダルではない）
- [ ] 変更は自動保存される

#### US-3.2: タスク削除
**As a** ユーザー  
**I want to** 不要になったタスクを削除できる  
**So that** タスクリストをクリーンに保てる

**受け入れ条件:**
- [ ] スワイプまたは削除ボタンで削除可能
- ~~[ ] 削除確認ダイアログ（オプションでスキップ可能）~~
- [ ] 削除取り消し機能（5秒間のUndo表示）

#### US-3.3: 締切設定
**As a** ユーザー  
**I want to** 重要なタスクに締切を設定できる  
**So that** 期限のあるタスクを確実に管理できる

**受け入れ条件:**
- [ ] カレンダーUIで日付選択
- [ ] 「今日」「明日」「来週」のクイック設定
- ~~[ ] 締切が近いタスクは視覚的に強調表示~~
- ~~[ ] 期限切れタスクは最優先で表示~~

#### US-3.4: 全タスク表示
**As a** ユーザー  
**I want to** 必要な時に全てのタスクを確認できる  
**So that** タスクの全体像を把握できる

**受け入れ条件:**
- [ ] 特別な操作（長押し、ジェスチャー等）で全タスク画面へ遷移
- [ ] 重さ、締切、作成日でソート可能
- [ ] 検索機能
- [ ] この画面では表示制限なし
- [ ] 基本画面へ簡単に戻れる
- [ ] 一括削除機能（チェックボックスを付けたタスクを全削除、全部チェックするボタンあり）


#### US-3.5: 完了タスク表示
**As a** ユーザー  
**I want to** 必要な時に完了タスクを確認できる  
**So that** 間違えて完了したタスクを未完了に復帰できる

**受け入れ条件:**
- [ ] 特別な操作（長押し、ジェスチャー等）で完了タスク画面へ遷移
- [ ] 重さ、締切、作成日でソート可能
- [ ] 検索機能
- [ ] 一括削除機能（チェックボックスを付けたタスクを全削除、全部チェックするボタンあり）
- [ ] 一括復帰機能
- [ ] 基本画面へ簡単に戻れる


### エピック4: システムの持続可能性

#### US-4.1: データの永続化
**As a** ユーザー  
**I want to** どのデバイスからでも同じタスクリストにアクセスできる  
**So that** 場所を選ばずタスク管理ができる

**受け入れ条件:**
- [ ] クラウドにデータが保存される
- [ ] 自動同期（リアルタイム）
- [ ] 同期エラー時の通知とリトライ

#### US-4.2: 認証とセキュリティ
**As a** ユーザー  
**I want to** 自分だけがアクセスできる  
**So that** プライベートな情報を安全に管理できる

**受け入れ条件:**
- [ ] シンプルな認証方法（マジックリンク類）
- [ ] セッション管理（JWT, 一定期間ログイン状態維持）
- [ ] HTTPS通信
- [ ] データの暗号化

#### US-4.3: レスポンシブデザイン
**As a** ユーザー  
**I want to** スマホでもPCでも快適に使える  
**So that** どんな状況でもタスク管理ができる

**受け入れ条件:**
- [ ] スマホ画面に最適化されたUI
- [ ] PC画面でも見やすいレイアウト
- [ ] タッチ操作とマウス操作の両対応
- [ ] 画面サイズに応じた表示タスク数の調整

#### US-4.4: パフォーマンス
**As a** ユーザー  
**I want to** サクサク動作する  
**So that** ストレスなく使い続けられる

**受け入れ条件:**
- [ ] 初回読み込み2秒以内
- [ ] タスク追加・完了は即座に反映
- [ ] LLM処理中も他の操作可能
- [ ] 1000タスクまでは問題なく動作
